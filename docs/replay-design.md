# 战斗看板系统设计

## 核心思路

组件化，三个独立面板可自由组合布局。不需要全屏大动画。

## 面板组成

### ③ ReplayPanel — 回放面板

紧凑型动画回放 + 同步日志：

```
┌────────────────────────────────┐
│  ▶ ⏸  1x  2x  4x              │  控制栏
├────────────────────────────────┤
│  [一刀]────●────────[奇岚]     │  横轴像素动画
│           3.0m                 │  (紧凑型，不需要大画面)
├────────────────────────────────┤

│    命中104%·51% 暴击44%·38%    │  同步日志

│  #2 雷暴 → 一刀               │
│    命中73%·12%                 │
└────────────────────────────────┘
```

## 布局系统

用 CSS Grid/Flexbox，面板通过 props 控制显示：

```tsx
// 可选布局示例：
<BattleViewer
  left={<BuildPanel character="qilan" />}
  center={<ReplayPanel events={log.getAll()} />}
  right={<BattlePanel state={snapshots} />}
/>

// 或紧凑布局：
<BattleViewer
  top={<ReplayPanel events={log.getAll()} compact />}
  bottom={<BuildPanel character="qilan" />}
/>
```

## 数据流

```
engine.state.log.getAll() ──→ ReplayPanel
                                      │
engine.state.characters  ──→ BattlePanel
engine.state.position    ──→ BattlePanel
engine.state.pendingBuffs──→ BattlePanel

Character.build ──→ BuildPanel
```

所有面板都是**纯展示组件**，数据通过 props 传入。

## 实施顺序

1. **BuildPanel** — 纯展示，最容易，直接从 Character 对象读数据
2. **ReplayPanel** — 核心，回放引擎 + Canvas 动画 + 同步日志
3. **BattlePanel** — 战斗状态快照，从 BattleState 读取

## 像素动画位置

Canvas 横轴只需要显示两个像素小人之间的相对位置，宽度 ~300-400px 足够：

```
[  一刀  ]──●────●──[  奇岚  ]
  左侧        距离    右侧
```

每个角色 16×16 像素，放大 2-3 倍显示。武器作为叠加层画在角色旁。
