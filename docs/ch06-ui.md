# 第 6 章：UI 与交互设计

---

## 6.1 设计语言

### 6.1.1 视觉风格

**复古终端/CRT 风**，以文字为主体，轻量像素级视觉点缀。

```
基调:
  ┌──────────────────────────────────────┐
  │  █████  ██████  DANTIAO  v0.1       │  ← 像素字体标题
  │  ──────────────────────────────────  │
  │  > 你站在一座废弃的道观前...         │  ← 等宽字体正文
  │  > 门缝里透出微弱的红光。            │
  │                                     │
  │  ┌─────────────────────────────┐    │
  │  │ [1] 推门而入  [2] 绕后      │    │  ← 选项卡（内容区内）
  │  │ [3] 仔细观察  [4] 离开      │    │
  │  └─────────────────────────────┘    │
  └──────────────────────────────────────┘
```

关键视觉元素：

- **底色**：纯黑（`#0a0a0a`），模拟关掉的 CRT 屏幕
- **主色**：磷光绿（`#33ff33`），模拟单色显示器
- **辅色**：白色（`#cccccc`）用于常规文字，琥珀色（`#ffb000`）用于高亮/稀有物品
- **字体**：等宽像素字体（`Press Start 2P` 用于标题，`Fira Code` / `JetBrains Mono` 用于正文）
- **边框**：单像素点阵边框，模拟 ASCII 艺术效果
- **装饰**：极少量 CRT 扫描线（CSS 伪元素 overlay），不干扰可读性
- **图标**：像素级 Unicode 符号或 CSS-drawn 小图标替代图片（`▶` `●` `◈` `◆` `■` `⚔` `🛡` `❤` `◆`）

### 6.1.2 色彩体系

| 用途        | 色值      | 说明           |
| ----------- | --------- | -------------- |
| 背景        | `#0a0a0a` | 纯黑背景       |
| 主文字      | `#33ff33` | 绿色主色调     |
| 次要文字    | `#cccccc` | 灰白辅助文字   |
| 高亮/稀有   | `#ffb000` | 琥珀色强调     |
| 危险/负面   | `#ff3355` | 红色警告       |
| 正面/增益   | `#33ff99` | 青绿增益       |
| 边框/分隔线 | `#335533` | 暗绿边框       |
| 焦点指示器  | `#33ff33` | 与主色一致     |
| 禁用/不可选 | `#335533` | 暗绿，低对比度 |

### 6.1.3 响应式策略

- **桌面端优先**（Min 1024×768，推荐 1280×720 以上）
- 布局使用 CSS Grid 三栏
- 未来移动端使用相同组件层，只调整排列方向（从上到下），Info/Log 面板可折叠
- 最小字号保证 16px，确保可读性

### 6.2.2 各区域说明

**Header Bar**

- 固定顶栏，始终可见
- 左侧：当前节点名称/标题
- 右侧：金币、HP 条（缩略）、设置按钮

**Info Panel（左栏）**

- 固定宽度 ~280px
- 分 Tab，始终可见，不随屏幕切换变化：
    - **属性** (Stats) — 6 维 + HP/AP/锻体等级
    - **功法** (Skills) — 已学功法列表，每个可展开详情
    - **招式** (Actions) — 已装备招式 + AP 消耗/效果简述
    - **奇物** (Artifacts) — 奇物 + 义体（部位/惩罚）+ 本命物
    - **触发器** (Trigger) — 已装备触发器 + 触发条件
- Tab 头始终可见，内容区域可滚动
- Hover/Focus 任何条目时展开 Tooltip 详情

**中央区域（上→中→下）**

分为三个纵向堆叠的子区域：

1. **节点信息区（顶）** — 当前屏幕的标题/上下文
    - 战斗：对手卡牌（名字、流派、基本信息）
    - 事件：事件标题 + 场景描述（精简）
    - 商店："黑市商店" + 店主一句话
    - 准备阶段："准备阶段" + 提示文字

2. **内容区（中）** — 核心交互区域，占用大部分垂直空间
    - 选卡：3 选 1 / 重要战斗单卡 / 奖励选择
    - 战斗：Canvas 动画（角色站位、动作播放）
    - 商店：商品列表（可选购）
    - 准备阶段：配置操作（装备/分配属性/打开匣子）
    - 选择可以悬浮在内容上（如奖励选择覆盖层）

3. **对话区（底）** — 叙事/日志实时文本，高度可调
    - 事件：叙事文本逐行显示
    - 战斗：战斗日志（每步操作一行）
    - 商店：商人对话
    - 可折叠（`H` 键或点击 `[−]`），折叠后只显示最后一行 + `[...N 条]`
    - 始终保持底部对齐，新内容自动滚到底部

**日志面板（右栏）**

- 固定宽度 ~240px
- **始终可见**，显示历史日志的摘要列表，按时间倒序
- 顶部过滤器 Tab：`全部` / `战斗` / `事件` / `系统`
- 每个条目一行摘要，可滚动
- 选中条目在内容区或浮层中展开详情
- `L` 键或点击底栏打开全屏日志视图

### 6.2.3 无固定选项栏

游戏**没有独立的 Options Bar**。所有交互都嵌入在内容区中：

- **3 选 1**：内容区显示三张卡片，点击/键盘选择
- **重要战斗**：内容区显示对手卡 + 开始按钮
- **准备阶段**：内容区显示配置面板
- **奖励选择**：悬浮在内容区上的选择卡片

## 6.6 Canvas 战斗可视化

### 6.6.1 布局

内容区中的 Canvas 尺寸由 CSS Grid 控制，自适应：

```
┌──────────────────────────────────┐
│  ┌──────┐          ┌──────┐     │
│  │玩家  │  6m       │对手  │     │
│  │ ██   │ ←──────→ │ ██   │     │
│  │ ██   │  距离     │ ██   │     │
│  └──────┘          └──────┘     │
│                                  │
│  HP ▓▓▓▓▓▓▓░░░ 140/220          │
│  AP ▓▓▓▓▓▓▓▓▓▓ 10/10            │
└──────────────────────────────────┘
```

### 6.6.2 元素

- **角色精灵**：像素风小人（站立/攻击/受击/移动 四帧）
- **距离指示**：角色之间的水平间距，实时更新
- **HP/AP 条**：角色下方
- **状态图标**：角色上方浮动（灼烧 🔥、中毒 ☠、眩晕 💫 等）
- **伤害数字**：命中时从目标位置向上飘动，1.5s 后消失
- **动作名称**：每次行动时在画面中央显示动作名

### 6.6.3 动画（MVP）

| 动作     | 动画                | 时长 |
| -------- | ------------------- | ---- |
| 普通攻击 | 角色前倾 → 目标闪烁 | 0.3s |
| 招式释放 | 角色闪白 → 目标闪烁 | 0.5s |
| 受击     | 角色抖动            | 0.2s |
| 移动     | 角色平滑平移        | 0.3s |
| 闪避     | 角色闪烁移动        | 0.3s |
| 格挡     | 角色前出现盾状光效  | 0.3s |
